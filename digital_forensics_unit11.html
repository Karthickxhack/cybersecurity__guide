179 🔍 Digital Forensics Intro
📖 Definition

Digital Forensics is the scientific process of identifying, preserving, extracting, analyzing, and documenting digital evidence from electronic devices for use in legal investigations. It answers: What happened? When? Who did it? How?

🌿 Branches of Digital Forensics
💽 Disk Forensics 🧠 Memory Forensics 🌐 Network Forensics 📱 Mobile Forensics ☁️ Cloud Forensics 📧 Email Forensics 🗄️ Database Forensics
⚖️ Purpose & Use Cases
Criminal prosecution Corporate investigation Data breach analysis Fraud detection Intellectual property theft Insider threat investigation
🔴 Real-Time Example
In 2022, a hospital employee was suspected of stealing patient records. Digital forensics investigators examined USB logs, browser history, and email attachments on their workstation — finding 14,000 patient records had been copied 3 weeks before their resignation. 🏥💾

✅ Advantages

  • Recovers deleted evidence
  • Provides court-admissible proof
  • Identifies attack vectors
  • Supports compliance audits
  • Helps prevent future incidents

❌ Disadvantages

  • Time-consuming & expensive
  • Evidence can be encrypted/wiped
  • Anti-forensics tools exist
  • Requires specialist expertise
  • Legal jurisdiction challenges

180 ⚙️ Forensic Process
📖 Definition

The Forensic Process is a structured, repeatable methodology used to investigate digital incidents while maintaining evidence integrity. Every step must be documented to ensure findings are legally admissible.

1️⃣ IDENTIFICATION
Identify incident scope, affected systems, evidence sources
2️⃣ PRESERVATION
Isolate systems, create forensic images, hash verification
3️⃣ COLLECTION
Gather evidence using write blockers, proper tools
4️⃣ EXAMINATION
Extract data, recover deleted files, parse artifacts
5️⃣ ANALYSIS
Correlate findings, reconstruct timeline, identify attacker
6️⃣ PRESENTATION
Write forensic report, present findings in court/to management
7️⃣ REVIEW
Lessons learned, improve security posture

181 🧪 Evidence Collection
📖 Definition

Evidence Collection in digital forensics is the process of gathering digital artifacts in a legally sound manner — ensuring data integrity, avoiding contamination, and following the Order of Volatility (most volatile → least volatile).

⚡ Order of Volatility (RFC 3227)

1. CPU Registers & Cache 🔥

Lost immediately when power off — nanoseconds lifespan. Collect first!

2. RAM / Main Memory 🧠

Volatile — lost on reboot. Contains running processes, passwords, encryption keys.

3. Network State 🌐

Active connections, ARP cache, routing tables — changes constantly.

4. Running Processes & Temp Files ⚙️

Temporary data, swap space, open file handles.

5. Hard Disk / SSD 💽

Persistent storage — survives power off. Collect using write-blocker + imaging.

6. Remote / Cloud Logs ☁️

May have retention policies — request quickly before logs roll over.

7. Archival Media 📼

Backups, optical disks — most stable, collected last.

# Create forensic disk image with dd (Linux) sudo dd if=/dev/sda of=/evidence/disk.img bs=512 conv=noerror,sync # if = input device (suspect drive) # of = output image file # bs = block size # conv=noerror = continue on read errors # Verify integrity with MD5 hash md5sum /dev/sda > original_hash.txt md5sum /evidence/disk.img > image_hash.txt diff original_hash.txt image_hash.txt # If output is BLANK → hashes MATCH → evidence intact ✅
Evidence TypeCollection MethodToolPitfall
💽 DiskBit-for-bit image with write blockerFTK Imager, ddNever write directly to original
🧠 RAMLive acquisition while system runningVolatility, DumpIt, WinPmemPowered-off = evidence lost
🌐 NetworkPacket captureWireshark, tcpdumpEncryption hides content

182 ⛓️ Chain of Custody
📖 Definition

Chain of Custody (CoC) is the documented, chronological record of who collected, handled, transferred, analyzed, and stored digital evidence — from discovery to courtroom. A broken chain = evidence inadmissible in court. ⚖️

🔍 Evidence Discovered at Scene
📋 Document: Who found it, when, where, condition
🔒 Seal & Label evidence with unique ID + hash
🚗 Transport to forensic lab — log every transfer
🔬 Analysis by examiner — document every action taken
🗄️ Secure storage — access log required
⚖️ Court presentation — CoC document proves integrity
CoC Document FieldPurposeExample
📋 Case NumberUnique investigation IDCASE-2024-0047
📅 Date/TimeWhen evidence was touched2024-03-15 14:32 UTC
👤 Handler NameWho touched the evidenceDet. J. Smith
📍 LocationWhere evidence wasSuspect's office, desk 3
#️⃣ Hash ValueIntegrity verificationSHA-256: a3f9...
🔒 Seal #Tamper-evidence seal IDSEAL-0094-X
✍️ SignaturesHandover acknowledgementCollector + Recipient sign
🔴 Real-Time Example
In a murder trial, police seized a suspect's laptop. The defense argued the hard drive was accessed without documentation between collection and analysis. Because no CoC log existed for a 3-hour gap, the digital evidence was ruled inadmissible. The case collapsed. ⚖️

✅ Why CoC Matters

  • Evidence legally admissible
  • Proves no tampering occurred
  • Accountability for each handler
  • Required by all legal systems

❌ What Breaks CoC

  • Undocumented access
  • Hash mismatch after handover
  • Improper storage conditions
  • Missing signatures on transfer

183 💽 Disk Forensics
📖 Definition

Disk Forensics is the analysis of hard drives, SSDs, USB drives, and other storage media to recover files, examine file systems, undelete data, and reconstruct user activity. It is the most common branch of digital forensics.

🔬 What Investigators Look For
🗑️ Deleted files 📧 Emails & attachments 🌐 Browser history 🔑 Password hashes 📂 File metadata ⏱️ File timestamps (MAC) 🔗 Link files (.lnk) 📦 Compressed archives
📐 Key Concepts

MAC Times: Modified · Accessed · Created timestamps

File Carving: Recovering files from unallocated space using file signatures (magic bytes)

Slack Space: Unused space in file clusters that may contain old data fragments

# Autopsy (CLI via Sleuth Kit) — list files on image fls -r /evidence/disk.img # List all files recursively icat /evidence/disk.img 1234 # Extract file by inode number # Find deleted files (marked with * asterisk) fls -r -d /evidence/disk.img # Output example: # r/r 1234: Documents/report.docx # r/r * 1235: Documents/secret.txt ← deleted file! # Recover deleted file icat /evidence/disk.img 1235 > recovered_secret.txt
ArtifactLocation (Windows)Forensic Value
🌐 Browser History%AppData%\Chrome\User DataVisited sites, search terms
📋 Prefetch FilesC:\Windows\Prefetch\Programs that were run
🔗 LNK Files%AppData%\Recent\Recently opened files
🗂️ RegistryC:\Windows\System32\config\USB history, user activity
📦 Recycle BinC:\$Recycle.Bin\Deleted files + timestamps
💤 HibernationC:\hiberfil.sysSnapshot of RAM contents

# Volatility 3 — memory forensics framework # Identify OS profile of memory dump python3 vol.py -f memory.dmp windows.info # List running processes python3 vol.py -f memory.dmp windows.pslist # PID PPID Name Offset # 4 0 System 0x... # 688 4 smss.exe 0x... # 1337 688 malware.exe 0x... ← suspicious! # Check network connections python3 vol.py -f memory.dmp windows.netstat # Dump malicious process memory python3 vol.py -f memory.dmp windows.memmap --pid 1337 --dump
🔴 Real-Time Example
Attackers used Meterpreter (fileless malware) — it ran entirely in RAM, leaving no traces on disk. Traditional antivirus found nothing. Memory forensics with Volatility revealed the injected shellcode in explorer.exe memory space and the attacker's C2 server IP. 🦠🧠

185 🌐 Network Forensics
📖 Definition

Network Forensics is the monitoring, capturing, storing, and analyzing of network traffic to detect intrusions, reconstruct attacks, identify data exfiltration, and gather evidence of network-based crimes.

🎯 What Network Forensics Reveals
🚨 Intrusion attempts 📤 Data exfiltration 🔗 C2 communications 🦠 Malware traffic 👤 Attacker IP addresses ⏱️ Attack timeline
⚔️ Catch-it-as-you-can vs Stop-look-listen

Catch-it-as-you-can: Capture ALL packets, analyze later (needs huge storage)

Stop-look-listen: Filter in real time, analyze on the fly (may miss some packets)

# tcpdump — capture packets on Linux tcpdump -i eth0 -w capture.pcap # Capture all traffic to file tcpdump -i eth0 host 192.168.1.5 # Filter by IP tcpdump -i eth0 port 443 # Filter HTTPS traffic tcpdump -i eth0 -r capture.pcap # Read captured file # Wireshark display filter examples http.request.method == "POST" # Show POST requests ip.src == 10.0.0.5 # Traffic from attacker IP dns.qry.name contains "malware.com" # DNS lookups to bad domain

# Linux — investigate failed SSH logins (brute force) grep "Failed password" /var/log/auth.log | tail -20 # Mar 15 03:21:44 sshd[1234]: Failed password for root from 192.168.1.100 port 55234 # Count attacks per IP address grep "Failed password" /var/log/auth.log | \ awk '{print $11}' | sort | uniq -c | sort -rn | head # 842 192.168.1.100 ← brute force attacker! # 12 10.0.0.5 # Windows Event ID cheat sheet: # 4624 = Successful logon # 4625 = Failed logon # 4648 = Logon with explicit credentials # 4688 = Process creation # 4663 = File access attempt # 4720 = User account created
🔴 Real-Time Example
A company noticed unusual activity at 2AM. Log analysis of Windows Security logs showed Event ID 4624 (logon) from a service account that had never logged in interactively before. Cross-referencing with firewall logs revealed data being sent to a Russian IP. Insider threat confirmed. 🕵️

188 🚨 Incident Response
📖 Definition

Incident Response (IR) is the organized approach to addressing and managing the aftermath of a security breach or cyberattack. The goal is to limit damage, reduce recovery time, and prevent recurrence. The NIST IR framework is the industry standard.

1️⃣ PREPARATION
IR plan, team, tools, playbooks ready before incident
2️⃣ IDENTIFICATION (Detection)
Detect incident via SIEM, IDS, user report
3️⃣ CONTAINMENT
Isolate affected systems — stop spread
4️⃣ ERADICATION
Remove malware, close vulnerabilities, patch systems
5️⃣ RECOVERY
Restore systems, verify integrity, monitor closely
6️⃣ LESSONS LEARNED
Post-incident report, improve defenses
PhaseKey QuestionsWho ActsOutput
1️⃣ PreparationAre we ready?CISO, IR TeamIR Plan, playbooks
2️⃣ IdentificationWhat happened?SOC AnalystIncident ticket
3️⃣ ContainmentStop the bleeding?IR TeamIsolated systems
4️⃣ EradicationIs threat gone?Forensics + IRClean systems
5️⃣ RecoveryBack to normal?IT + IR TeamRestored services
6️⃣ Lessons LearnedHow do we improve?All teamsPost-mortem report

✅ Good IR Outcomes

  • Minimal data loss
  • Fast return to operations
  • Legal evidence preserved
  • Attacker identified
  • Stronger future defenses

❌ Poor IR Consequences

  • Evidence destroyed by cleanup
  • Attacker persists undetected
  • Extended downtime & costs
  • Regulatory fines (GDPR)
  • Reputational damage

189 🕵️ Threat Intelligence
📖 Definition

Threat Intelligence (TI) is evidence-based knowledge about existing or emerging threats — including context, mechanisms, indicators, and actionable advice — that enables organizations to make informed decisions about defending against attacks.

📊 Types of Threat Intelligence

🔵 Strategic TI

High-level trends for executives. "APT groups targeting healthcare sector."

🟡 Operational TI

Details of specific campaigns. "Group X uses spear-phishing + Cobalt Strike."

🔴 Tactical TI

TTPs (Tactics, Techniques, Procedures). MITRE ATT&CK mappings.

🟢 Technical TI

IOCs: malware hashes, IPs, domains. Immediately actionable in tools.

190 🚩 IOC – Indicators of Compromise
📖 Definition

Indicators of Compromise (IOCs) are pieces of forensic data — such as file hashes, IP addresses, domain names, or registry keys — that identify potentially malicious activity on a system or network. They are the "fingerprints" left by attackers.

IOC TypeExampleWhere to FindHow Used
#️⃣ File Hash (MD5/SHA)a1b2c3d4e5f6...Disk, memory dumpsVirusTotal lookup
🌐 IP Address185.220.101.5Network logs, pcapBlock in firewall
🔗 Domain / URLmalware-c2.ruDNS logs, proxy logsBlock in DNS/proxy
📧 Email Addressphish@evil.comEmail server logsBlacklist sender
🗝️ Registry KeyHKLM\Run\evilappWindows RegistryIndicates persistence
📂 File Path/NameC:\Temp\svchost32.exeDisk, prefetchSearch & remove
🌍 User Agent StringMozilla/5.0 (custom)Web server logsBlock in WAF
🔑 Mutex NameGlobal\\EvilMutex123Memory analysisDetects running malware
🔺 IOC vs IOA

IOC (Indicator of Compromise): What happened — after-the-fact evidence. "We found malware hash on disk."

IOA (Indicator of Attack): What is happening — behavior-based. "Process is performing privilege escalation RIGHT NOW."

🏔️ Pyramid of Pain

Describes how hard it is for attackers to change each IOC type:

🔴 TTPs (hardest) → 🟠 Tools → 🟡 Network/Host Artifacts → 🟢 Domain Names → 🔵 IP Addresses → ⚪ Hash Values (easiest to change)

# Check file hash against VirusTotal (Python) import requests api_key = "YOUR_VT_API_KEY" file_hash = "d41d8cd98f00b204e9800998ecf8427e" url = f"https://www.virustotal.com/api/v3/files/{file_hash}" headers = {"x-apikey": api_key} response = requests.get(url, headers=headers) data = response.json() # Check detection count detections = data["data"]["attributes"]["last_analysis_stats"] print(f"Malicious: {detections['malicious']}/72 engines")

191 📁 Case Study – SolarWinds Supply Chain Attack (2020)
📖 Overview

The SolarWinds Sunburst attack was one of history's most sophisticated cyber espionage campaigns. Russian state-sponsored hackers (APT29/Cozy Bear) compromised SolarWinds' build process, inserting malware into the Orion software update — affecting 18,000+ organizations including US government agencies.

🏭 Attacker compromises SolarWinds build environment (Oct 2019)
☠️ SUNBURST malware injected into Orion update (March 2020)
📦 18,000+ organizations install trojanized update
⏳ Malware dormant for 12-14 days (evades sandboxes)
🌐 Beacons to C2 server via legitimate-looking HTTPS traffic
👤 Manual attacker selects high-value targets for espionage
🔍 Discovered by FireEye (Dec 2020) — IOCs shared globally
💥 US Treasury, DoD, DHS, 100+ private companies breached
🔬 Forensic Findings
  • Malware used DGA (Domain Generation Algorithm) for C2
  • Traffic mimicked legitimate SolarWinds traffic
  • Attackers used "living off the land" techniques
  • Memory-only payloads to evade disk forensics
  • Lateral movement via golden SAML tokens
✅ Lessons Learned
  • Secure the software supply chain
  • Code signing verification is critical
  • Zero-trust architecture needed
  • IOC sharing (ISAC) saves others
  • Memory forensics is essential

🎯 Unit 11 Conclusion

Digital Forensics is the backbone of cyber justice. Without it, attackers operate in the shadows, leave no accountability, and strike again. The core insight of this unit: every click, every file, every network packet leaves a trace — forensics finds it.

Master the forensic process, preserve evidence properly, maintain chain of custody, use the right tools, and share threat intelligence — and you'll be able to answer the three most important questions after any attack: What happened? Who did it? How do we stop it from happening again? 🔍⚖️
