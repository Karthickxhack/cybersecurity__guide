Introduction to Security Tools

⚙️
⚡ Summary
Security tools are software programs that help security professionals detect, analyse, prevent, and respond to threats — spanning offensive (pen testing) to defensive (monitoring, SIEM) categories. Choosing the right tool for the right task is the core skill.

Offensive vs Defensive Tools

⚔️ Offensive Tools (Red Team / Pen Test)

  • Used to simulate attacks and find vulnerabilities
  • Require explicit written authorisation
  • Examples: Metasploit, Burp Suite, Hydra
  • Goal: find weaknesses before real attackers
  • Used by: pen testers, red teams, bug hunters

🛡️ Defensive Tools (Blue Team / SOC)

  • Used to detect, monitor, and respond to attacks
  • Always authorised by the organisation
  • Examples: Splunk, ELK Stack, Wireshark, EDR
  • Goal: detect and stop attacks in real time
  • Used by: SOC analysts, incident responders, sysadmins

Tool Selection Framework

CHOOSING THE RIGHT TOOL
Task → Tool Selection Decision Tree:

"I need to scan a network for open ports"         → nmap
"I need to find web app vulnerabilities"          → Burp Suite + sqlmap
"I need to exploit a known CVE"                   → Metasploit
"I need to analyse suspicious network traffic"    → Wireshark + Zeek
"I need to search security logs for an incident"  → Splunk / ELK
"I need to crack a password hash"                 → Hashcat (GPU) / John (CPU)
"I need to scan for CVEs across my infrastructure" → Nessus / Qualys / OpenVAS
"I need to test WiFi security"                    → Aircrack-ng
"I need to gather OSINT on a target"              → theHarvester + Maltego
"I need to automate security checks"              → Python + Ansible + custom scripts

Evaluation criteria:
1. Open source vs commercial (cost vs support)
2. CLI vs GUI (scripting vs ease of use)
3. Active vs passive (detectable vs undetectable)
4. Real-time vs offline (live vs recorded data)
5. Legal requirements for your jurisdiction
📌 Conclusion

No single tool does everything. A cybersecurity professional needs to know the right tool for each job — and more importantly, understand what each tool does under the hood. Tools are enablers of understanding, not substitutes for it.

ip.src == 192.168.1.50                # Source IP only
tcp.port == 80                        # HTTP traffic
http                                  # All HTTP packets
http.request.method == "POST"         # POST requests only
http contains "password"              # HTTP with "password" in body
dns.qry.name contains "evil"          # Suspicious DNS queries
tcp.flags.syn == 1 && tcp.flags.ack == 0  # SYN packets (port scans)

frame.len > 1000                      # Large packets only
tls                                  # All TLS/SSL traffic
ssl.record.content_type == 23         # TLS application data
tcp.stream eq 5                      # Follow specific TCP stream
icmp.type == 8                        # ICMP echo requests (ping)
smb || smb2                          # SMB protocol

SIEM Basics

📊
⚡ Summary
SIEM (Security Information and Event Management) collects and correlates logs from all security tools into one platform — turning millions of individual log lines into actionable security alerts by detecting attack patterns across the full kill chain.
Definition
SIEM combines Security Information Management (SIM) (long-term log storage and compliance) with Security Event Management (SEM) (real-time threat detection and alerting). The result is a centralised platform that ingests logs from every source, correlates them against detection rules, and creates alerts for the SOC team.

How SIEM Works

SIEM DATA FLOW

     │                                                              │

Network
Cloud APIs

Example correlation rule (detect brute force):
IF event_type = "authentication_failure"
AND count(event) > 5
WITHIN 60 seconds
FROM same source_ip
THEN create alert: "BRUTE FORCE DETECTED" severity=HIGH

Multi-event correlation (detect lateral movement):
IF workstation_A fails SSH to workstation_B (auth failure)
AND workstation_A connects to workstation_C (SMB success)
AND workstation_A connects to DC via RPC
WITHIN 30 minutes
THEN alert: "LATERAL MOVEMENT DETECTED"

━━━ BASIC SEARCH SYNTAX ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# Search for specific terms:

━━━ SPL COMMANDS (PIPE-BASED LIKE UNIX) ━━━━━━━━━━━━━━━━━━━━━━
# Count events by source IP (find top talkers):
index=firewall action=blocked
| stats count by src_ip
| sort -count
| head 10

# Detect brute force (5+ failed logins from same IP in 5 minutes):
index=auth_logs "authentication failure"
| bin _time span=5m
| stats count by _time, src_ip
| where count > 5
| sort -count

# Detect after-hours logins (outside 8am-6pm):
index=auth_logs action=success
| eval hour=strftime(_time, "%H")
| where hour < 8 OR hour > 18
| table _time, user, src_ip, host

# Find PowerShell execution (fileless malware indicator):
index=windows_sysmon EventCode=1
| where match(CommandLine, "(?i)(powershell|pwsh)")
| where match(CommandLine, "(?i)(-enc|-encoded|-w hidden|-nop)")
| table _time, Computer, User, CommandLine

# Track data exfiltration (large outbound transfers):
index=network bytes_out > 100000000
| stats sum(bytes_out) as total_bytes by src_ip, dest_ip
| where total_bytes > 1000000000
| sort -total_bytes

# Correlate firewall blocks with internal scanner (lateral movement):
index=firewall action=blocked dest_port=445
| stats count by src_ip
| where count > 50
| rename src_ip as lateral_movement_source

Password Cracking Tools

🔓
⚡ Summary
Password cracking tools recover plaintext passwords from hash values using dictionary attacks, brute force, or rule-based mutations — Hashcat uses GPU acceleration for billions of guesses per second; John the Ripper is the CPU-based classic.
HASHCAT + JOHN — COMPLETE REFERENCE
━━━ HASHCAT (GPU-based — FASTEST) ━━━━━━━━━━━━━━━━━━━━━━━━━━━━

# Common hash modes:
-m 0      MD5                -m 1000   NTLM (Windows)
-m 100    SHA1               -m 1800   SHA-512 (Linux shadow)
-m 1400   SHA2-256           -m 3200   bcrypt
-m 5600   NTLMv2 (Net-NTLMv2) -m 13100 Kerberoast (TGS)
-m 22000  WPA2               -m 7400  SHA256crypt

# Attack modes:
┌──(kali㉿kali)-[~]
└─$ hashcat -m 0 -a 0 hash.txt rockyou.txt        # Dictionary attack (MD5)
┌──(kali㉿kali)-[~]
└─$ hashcat -m 1000 -a 0 ntlm.txt rockyou.txt -r best64.rule  # Rules (NTLM)
┌──(kali㉿kali)-[~]

┌──(kali㉿kali)-[~]
└─$ hashcat -m 22000 -a 0 wpa.hccapx rockyou.txt  # WPA2 cracking
┌──(kali㉿kali)-[~]

┌──(kali㉿kali)-[~]

# Mask attack (charset):
?l = lowercase (a-z)    ?u = uppercase (A-Z)
?d = digits (0-9)       ?s = special chars

# Automated subdomain scanner + vulnerability check:
import subprocess, requests

subdomains = subprocess.check_output(
    ['subfinder', '-d', 'target.com', '-silent']
).decode().strip().split('\n')

for sub in subdomains:
    try:
        r = requests.get(f'https://{sub}', timeout=5, verify=False)
        # Check for security headers
        if 'Content-Security-Policy' not in r.headers:
            print(f'[MISSING CSP] {sub}')
        if 'X-Frame-Options' not in r.headers:
            print(f'[CLICKJACK RISK] {sub}')
    except: pass

# Automated hash cracking pipeline:
import subprocess

hashes = open('hashes.txt').readlines()
for h in hashes:
    result = subprocess.run(
        ['hashcat', '-m', '1000', h.strip(), 'rockyou.txt', '--quiet'],
        capture_output=True, text=True
    )
    if ':' in result.stdout:
        print(f'[CRACKED] {result.stdout.strip()}')
