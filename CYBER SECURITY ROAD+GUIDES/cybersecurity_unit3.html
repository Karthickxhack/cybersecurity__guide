Topic 33

Introduction to Networking

🌐

Definition

Definition
A computer network is a collection of two or more devices connected to share resources, exchange data, and communicate. Networking is the foundation of all internet-connected cybersecurity — if you don't understand how data travels, you cannot defend it.

Why Networking Matters in Cybersecurity

📡
Data in Transit
All attacks on data-in-transit require network knowledge: sniffing, MitM, DDoS all target the network layer.
🗺️
Reconnaissance
Attackers map networks first. Understanding topology reveals what an attacker learns from a port scan.
🔥
Firewall Rules
Firewalls work at network/transport layers. Writing effective rules requires deep networking knowledge.
🕵️
Intrusion Detection
IDS/IPS analyse packet headers and payloads — impossible without understanding packet structure.
🔒
VPN & Encryption
VPNs create encrypted tunnels across untrusted networks — requires understanding routing and encapsulation.
🩺
Incident Response
Tracking an attacker through network logs requires knowing what normal vs abnormal traffic looks like.

Key Networking Terminology

TermDefinitionSecurity Relevance
NodeAny device on a network (PC, phone, printer, server)Every node is a potential attack entry point
LinkPhysical or logical connection between nodesWireless links easier to intercept than wired
BandwidthMaximum data capacity of a link (Mbps/Gbps)DDoS exhausts bandwidth to deny service
LatencyDelay for data to travel from A to B (ms)High latency = detection delay for IDS alerts
PacketUnit of data with header (routing info) + payload (data)Packet injection attacks forge packet headers
ProtocolSet of rules governing communication between devicesProtocol flaws = vulnerabilities (e.g. ARP has no authentication)
TopologyPhysical or logical arrangement of network nodesDetermines blast radius of a single compromised node
🔬 Exercise
  1. List three devices in your home that are network nodes. For each, identify what data they transmit.
  2. Why is understanding network topology important for an attacker performing reconnaissance?
📌 Conclusion

Every cybersecurity career path requires networking knowledge. Whether you want to be a penetration tester, SOC analyst, or security architect — you must understand how data moves before you can protect or attack it.

LAN vs WAN Comparison

🏠 LAN (Local Area Network)

  • Privately owned and managed
  • High speed, low latency
  • Shared resources: printers, NAS, servers
  • Uses Ethernet (wired) or WiFi (wireless)
  • Typical range: 1–500 metres
  • Security: internal threats, ARP attacks, rogue devices

🌍 WAN (Wide Area Network)

  • Spans cities, countries, or globally
  • Usually owned by ISPs/telecoms
  • Lower speed, higher latency than LAN
  • Uses leased lines, fibre, satellite, cellular
  • The internet is the largest WAN
  • Security: ISP-level attacks, BGP hijacking, interception

Network type determines the attack surface and available defences. LANs allow internal pivot attacks once one device is compromised. WANs add routing complexity and ISP-level threats. WiFi networks require WPA3 and strong passphrases.

Topic 35

Network Topologies

🔗

Physical vs Logical Topology

💡
Physical topology = how cables/devices are physically connected. Logical topology = how data actually flows, which may differ (e.g. Ethernet star topology physically, but bus logically at lower layers). Both matter for security analysis.

Five Core Topologies

⭐ Star
[PC1] \ [PC2]—[HUB/SW]—[PC3] / [PC4]
All devices connect to central switch/hub. Most common in modern networks. Failure of one device doesn't affect others, but hub/switch failure = total outage. Security: switch provides isolation; hub broadcasts everything to all ports (easy to sniff).
🚌 Bus
[PC1]─[PC2]─[PC3]─[PC4] └──────────────────┘ Shared backbone
All devices share a single cable. Legacy (coaxial). One cable break = entire network down. Security risk: every device sees ALL traffic — easy to sniff. Used in original Ethernet (10BASE2).
💍 Ring
[PC1]→[PC2] ↑ ↓ [PC4]←[PC3] Data flows one direction
Each device connects to exactly two others. Data passes around ring until it reaches destination. Token Ring (IBM). One break = entire ring fails. Not used in modern LANs but seen in SONET fibre rings (telecoms).
🕸️ Mesh
[A]───[B] │╲ /│ │ X │ │/ \│ [C]───[D] Full: every node connected
Every device connects to every other. Full mesh = n(n-1)/2 links. Highest redundancy, no single point of failure. Used in military/critical networks. Partial mesh common in WANs. Security: harder to isolate a breach.
🌳 Tree (Hierarchical)
[Core SW] / \ [Dist1] [Dist2] / \ / \ [A][B] [C][D]
Hierarchical star-of-stars. Modern enterprise standard (core/distribution/access layers). Easy to expand, segment, and manage. Security: VLANs at distribution layer. Core failure = total outage — must be redundant.
🌐 Hybrid
Real-world networks combine topologies (star+mesh for redundancy, tree+ring, etc.)
Combination of two or more topologies. Most real-world enterprise networks are hybrid — star topology within buildings, mesh between datacenters, tree hierarchy overall.

Topology Advantages & Disadvantages

TopologyAdvantagesDisadvantagesTypical Use
StarEasy troubleshoot; isolate faults; scalableSingle point of failure (central switch)Home/office LAN
BusSimple, cheap, easy to add devicesCollision domain; one break = total failureLegacy/industrial
RingEqual access; no collisions (token)Break = failure; slow (wait for token)Legacy Token Ring, SONET
MeshMaximum redundancy; no single failure pointExpensive; complex; many cablesWAN backbone, military
TreeScalable; easy to manage; segment trafficCore switch = SPF; hierarchical complexityEnterprise LAN
🔬 Exercise
  1. A company has 500 employees in one building. Which topology would you recommend and why?
  2. In a star topology using a hub vs a switch, which makes network sniffing easier for an attacker? Why?
  3. Draw a simple hybrid topology for a company with two offices (Star locally, connected via WAN mesh).
📌 Conclusion

Topology shapes both performance and security. Modern enterprise networks use tree/hierarchical topologies with mesh redundancy at the core. Understanding topology lets you identify chokepoints, single points of failure, and where to place security controls like firewalls and IDS sensors.

Topic 36

OSI Model Overview

📚

What is the OSI Model?

Definition
The OSI (Open Systems Interconnection) Model is a conceptual framework developed by ISO in 1984 that divides network communication into 7 distinct layers. It is a reference model — no real protocol implements all 7 layers exactly, but it provides a universal language for describing network behaviour.

The 7 Layers — Overview

7ApplicationUser-facing services: HTTP, FTP, SMTP, DNS🎯 XSS, SQLi, phishing
6PresentationData format, encryption/decryption, compression🎯 SSL strip, encoding attacks
5SessionSession establishment, management, termination🎯 Session hijacking
4TransportTCP/UDP, ports, segmentation, flow control🎯 SYN flood, port scan
3NetworkIP addressing, routing, fragmentation🎯 IP spoofing, ICMP flood
2Data LinkMAC addresses, Ethernet frames, switches🎯 ARP spoofing, MAC flood
1PhysicalBits as signals: cables, WiFi, fibre🎯 Wire tap, jamming, evil twin

Memory Tricks

MNEMONICS
# Top-down (7 to 1):
"All People Seem To Need Data Processing"
 Application  Presentation  Session  Transport  Network  Data Link  Physical

# Bottom-up (1 to 7):
"Please Do Not Throw Sausage Pizza Away"
 Physical  Data Link  Network  Transport  Session  Presentation  Application

# Cybersecurity layer-to-attack mapping:
Layer 1 (Physical)    → Wiretapping, jamming, hardware keylogger
Layer 2 (Data Link)   → ARP poisoning, MAC flooding, VLAN hopping
Layer 3 (Network)     → IP spoofing, routing attacks, ICMP attacks
Layer 4 (Transport)   → SYN flood, UDP flood, port scanning
Layer 5 (Session)     → Session hijacking, replay attacks
Layer 6 (Presentation)→ SSL strip, encryption downgrade
Layer 7 (Application) → XSS, SQLi, CSRF, phishing, RCE

Protocol Data Unit (PDU) names: L7–L5 = Data | L4 = Segment (TCP) / Datagram (UDP) | L3 = Packet | L2 = Frame | L1 = Bit. Each layer adds its own header during encapsulation, and strips it during decapsulation at the receiving end.
🔬 Exercise
  1. At which OSI layer does a switch operate? What about a router? A firewall? A proxy server?
  2. A Wireshark capture shows an unusual number of TCP SYN packets from one IP with no SYN-ACK responses. Which layer is being attacked and what attack is this?

📌 Conclusion

The OSI model is the cybersecurity professional's universal map. Every attack, every defence, every tool maps to a layer. When analysing an incident, always ask: "At which layer is this occurring?" — that determines your tools and countermeasures.

Layer 4 Detail: TCP 3-Way Handshake

TCP HANDSHAKE + SYN FLOOD
# Normal TCP connection establishment:
Client                          Server
  │──── SYN (seq=1000) ────────▶│   "I want to connect"
  │◀─── SYN-ACK (seq=2000) ─────│   "OK, I'm ready"
  │──── ACK (seq=2001) ─────────▶│   "Connection established!"
  │═══════ DATA FLOWS ══════════│

# SYN Flood Attack:
Attacker (spoofed IPs)          Server
  │──── SYN (src=1.2.3.4) ─────▶│  Server allocates half-open connection
  │──── SYN (src=5.6.7.8) ─────▶│  Server allocates half-open connection
  │──── SYN (src=9.10.11.12) ──▶│  Server table fills up...
  │──── SYN × millions ─────────▶│  Server can't accept new connections!
                                  │  Legitimate users get REFUSED

# Defence: SYN Cookies
Server doesn't allocate memory until ACK received — encodes state in ISN

Layer 2 Detail: ARP & MAC Flooding

LAYER 2 ATTACKS
# Normal switch operation:
Switch maintains MAC Address Table (CAM table):
Port 1 → MAC: AA:BB:CC:DD:EE:01
Port 2 → MAC: AA:BB:CC:DD:EE:02
Switch sends frames ONLY to correct port (unicast)

# MAC Flooding Attack (macof tool):
Attacker floods switch with 100,000s of fake MAC addresses
CAM table overflows → switch can't find entries
Switch "fails open" → broadcasts ALL frames to ALL ports
= Attacker can now sniff all traffic on the switch!

# Defence: Port security (limit MACs per port), 802.1X authentication

# VLAN Hopping (Double Tagging):
Attacker sends frame with TWO 802.1Q VLAN tags
First switch strips outer tag → second switch uses inner tag
Attacker reaches VLAN they shouldn't have access to!
# Defence: Set native VLAN to unused VLAN, disable trunk on access ports
🔬 Exercise
  1. Explain the TCP 3-way handshake. What specifically makes a SYN flood attack effective against unprotected servers?
  2. What is a VLAN, and how does VLAN hopping bypass network segmentation?
📌 Conclusion

Each OSI layer has its own protocol-specific vulnerabilities. Defenders must understand attacks at every layer because a network perimeter firewall (Layer 3/4) provides zero protection against Layer 7 attacks like XSS or Layer 2 attacks like ARP spoofing inside the network.

IP Addressing (IPv4 & IPv6)

🔢

IPv4 Addressing

IPv4 STRUCTURE
# IPv4: 32-bit address, written in dotted decimal
192.168.10.25
│   │   │  └─ Host part (identifies device on subnet)
│   │   └──── Subnet part (depends on mask)
│   └──────── Network part
└──────────── Network part

# Each octet = 8 bits → range 0-255
192       .168       .10        .25
11000000 . 10101000 . 00001010 . 00011001

# Special IPv4 addresses:
0.0.0.0          → This host on this network
127.0.0.1        → Loopback (localhost) — test local TCP/IP
169.254.x.x      → APIPA (auto-assigned when DHCP fails)
255.255.255.255  → Limited broadcast
x.x.x.255        → Directed broadcast (network-specific)

# Private address ranges (RFC 1918) — NOT routable on internet:
10.0.0.0/8         → 10.0.0.0 – 10.255.255.255    (16.7M hosts)
172.16.0.0/12      → 172.16.0.0 – 172.31.255.255   (1M hosts)
192.168.0.0/16     → 192.168.0.0 – 192.168.255.255  (65K hosts)

IPv6 Addressing

IPv6 STRUCTURE
# IPv6: 128-bit address, written in hex with colons
2001:0db8:85a3:0000:0000:8a2e:0370:7334
└─────────────────────────────────────────┘
          8 groups of 4 hex digits

# Shorthand rules:
Leading zeros in a group can be omitted:
  0db8 → db8,  0000 → 0

Consecutive all-zero groups can be replaced ONCE with ::
  2001:0db8:0000:0000:0000:0000:0370:7334
  → 2001:db8::370:7334

# Special IPv6 addresses:
::1                   → Loopback (same as 127.0.0.1)
fe80::/10             → Link-local (non-routable, like APIPA)
fc00::/7              → Unique local (like RFC 1918 private)
ff02::1               → All nodes multicast
2000::/3              → Global unicast (public internet)

# Security note — IPv6 autoconfiguration (SLAAC):
Devices can self-assign IPv6 without DHCP using MAC address
→ Privacy concern: MAC embedded in IPv6 address!
→ Use Privacy Extensions (RFC 4941) to randomise

IPv4 vs IPv6 Comparison

FeatureIPv4IPv6
Address size32-bit (~4.3 billion)128-bit (340 undecillion)
Address formatDotted decimal: 192.168.1.1Hex colon: 2001:db8::1
ConfigurationManual or DHCPSLAAC or DHCPv6
NAT required?Yes (address exhaustion)No (vast address space)
Built-in securityNo (IPSec optional)IPSec mandatory (though often disabled)
Header size20-60 bytes (variable)40 bytes (fixed) — faster routing
BroadcastYes (network overhead)No — uses multicast instead
Adoption (2024)~67% of traffic~40% of traffic (growing)

Topic 40

Subnetting Basics

🧮

What is Subnetting?

Definition
Subnetting divides a large IP network into smaller sub-networks (subnets). This improves performance (reduces broadcast domains), security (isolates segments), and enables efficient IP allocation. Security teams use subnets to isolate sensitive systems (DMZ, finance, IoT) from each other.

Subnet Mask & CIDR Notation

SUBNETTING
# Subnet mask: 32 bits — 1s = network portion, 0s = host portion
255.255.255.0  = 11111111.11111111.11111111.00000000 = /24

# CIDR notation: /prefix = number of 1s in subnet mask
192.168.1.0/24
Network:   192.168.1.   (first 24 bits)
Hosts:     .0 – .255    (last 8 bits)
Usable:    192.168.1.1 – 192.168.1.254  (254 hosts)
Network:   192.168.1.0  (reserved)
Broadcast: 192.168.1.255 (reserved)

# Common subnet sizes:
Prefix  Subnet Mask      Hosts    Use Case
/30     255.255.255.252    2      Point-to-point router links
/29     255.255.255.248    6      Small office
/28     255.255.255.240   14      Small segment
/27     255.255.255.224   30      Medium segment
/24     255.255.255.0    254      Standard office LAN
/23     255.255.254.0    510      Larger LAN
/22     255.255.252.0   1022      Campus LAN
/16     255.255.0.0    65534      Large enterprise
/8      255.0.0.0     16M+       ISP block

# Formula:
Usable hosts = 2^(32-prefix) - 2
Number of subnets = 2^(bits borrowed from host)

# Security subnetting example:
Corporate network: 10.0.0.0/8
  10.1.0.0/24  → DMZ (web servers — internet-facing)
  10.2.0.0/24  → Finance (restricted, firewall rules)
  10.3.0.0/24  → IT/Admin (heavily protected)
  10.4.0.0/22  → Staff workstations (1022 hosts)
  10.10.0.0/24 → IoT devices (isolated — no internet access!)

Security Benefits of Subnetting

Security BenefitHow Subnetting HelpsExample
IsolationBreached subnet can't directly reach others without passing firewallIoT subnet can't reach finance subnet
Reduced broadcast domainBroadcast storms/attacks stay within subnetARP flood on WiFi subnet doesn't affect servers
Traffic monitoringRouter logs inter-subnet traffic, making lateral movement visiblePC pinging servers in server subnet triggers alert
ACL applicationApply access control lists at subnet boundariesOnly admin subnet can SSH to server subnet
DMZ architectureWeb servers in separate subnet from internal networkCompromised web server can't reach database directly
🔬 Exercise
  1. A company is assigned 192.168.5.0/24. They need 4 equal subnets. Calculate the new prefix length, subnet mask, and usable host range for each subnet.
  2. What is the network address, broadcast address, and number of usable hosts for 10.20.30.0/27?
  3. Why should IoT devices always be placed in a separate, isolated subnet?
📌 Conclusion

Subnetting is a fundamental security tool — it's the network equivalent of putting valuables in a safe within a house. A well-segmented network contains breaches, enables targeted monitoring, and limits an attacker's lateral movement. Every security architecture begins with proper network segmentation.

DNS Security — DNSSEC & DoH

DNS SECURITY
# DNSSEC — adds digital signatures to DNS records
DNS resolver verifies signature using public key
→ Attacker cannot forge signed records
→ Protects against cache poisoning and MitM DNS attacks
→ Deployed on ~17% of domains (growing slowly)

# DNS over HTTPS (DoH) — RFC 8484
Traditional DNS: PLAINTEXT on UDP port 53 → ISP/attacker sees all queries
DoH:             ENCRYPTED in HTTPS on port 443 → queries look like web traffic
Providers: Cloudflare (1.1.1.1), Google (8.8.8.8), NextDNS
Used by: Firefox, Chrome, Windows 11 (DoH by default)

# DNS over TLS (DoT) — RFC 7858
Similar to DoH but uses TLS on port 853
Less stealthy than DoH (port 853 can be blocked)

# DNS Recon tools (penetration testing):
dig example.com ANY       → All DNS records
dig -x 93.184.216.34      → Reverse lookup
fierce -dns example.com   → Subdomain brute force
dnsx -l domains.txt       → Bulk DNS resolution
amass enum -d example.com → Comprehensive subdomain enum

DNS is critical infrastructure and a significant attack vector. Enable DNSSEC for your domains, use DoH/DoT for privacy, monitor DNS queries for tunnelling, and audit subdomain records regularly. Control of DNS = control of your internet identity.

What is a Port?

Definition
A port is a 16-bit logical endpoint (0–65535) at the transport layer that identifies which application/service should receive network traffic. Think of an IP address as a building address and a port as the specific apartment number. IP address gets you to the server; port gets you to the right service.

ARP & MAC Address

🪪

MAC Address

Definition
A MAC (Media Access Control) address is a 48-bit hardware identifier permanently assigned to a network interface by its manufacturer. It operates at Layer 2 (Data Link) and is used for communication within a single network segment (LAN). Format: AA:BB:CC:DD:EE:FF (6 bytes in hex).
MAC ADDRESS ANATOMY
AA:BB:CC : DD:EE:FF
└────────┘   └────────┘
  OUI          Device Identifier
(Manufacturer) (Unique per device)

# OUI lookup reveals manufacturer:
00:1A:2B → Apple, Inc.
00:50:56 → VMware (virtual machine — attacker tip!)
08:00:27 → VirtualBox (virtual machine)
B8:27:EB → Raspberry Pi Foundation

# MAC address security notes:
→ MAC addresses can be SPOOFED in software:
  ip link set dev eth0 address 00:11:22:33:44:55
→ MAC filtering on WiFi = false security (easily bypassed)
→ MAC addresses visible in WiFi probe requests = privacy leak
→ Modern devices randomise MAC for WiFi scanning (iOS, Android, Win11)

ARP Protocol & ARP Spoofing

Definition
ARP (Address Resolution Protocol) translates IP addresses to MAC addresses within a LAN. When a device wants to send data to 192.168.1.5, it broadcasts "Who has 192.168.1.5?" and the device with that IP replies "I do — my MAC is AA:BB:CC:DD:EE:FF". ARP has no authentication — anyone can reply.
ARP SPOOFING ATTACK
# Normal ARP:
Alice (192.168.1.10)  →  broadcasts: "Who has 192.168.1.1? Tell 192.168.1.10"
Router (192.168.1.1)  →  unicast: "192.168.1.1 is at AA:BB:CC:11:22:33"
Alice caches: 192.168.1.1 → AA:BB:CC:11:22:33

# ARP Spoofing (arpspoof/ettercap):
Attacker sends unsolicited ARP replies (Gratuitous ARP):
To Alice:  "192.168.1.1 (router) is at MY MAC (attacker)"
To Router: "192.168.1.10 (Alice) is at MY MAC (attacker)"

Alice's ARP cache: 192.168.1.1 → ATTACKER MAC (poisoned!)
Router's ARP cache: 192.168.1.10 → ATTACKER MAC (poisoned!)

Result: All Alice↔Router traffic flows through attacker!
Attacker can: read, modify, or drop all traffic

# Defence:
Dynamic ARP Inspection (DAI)  → Switch validates ARP against DHCP snooping table
Static ARP entries            → Hardcode gateway MAC (impractical at scale)
XDR/SIEM monitoring           → Detect ARP table changes / duplicate IPs
802.1X                        → Only authenticated devices on network
🔬 Exercise
  1. Describe how an ARP spoofing attack enables a Man-in-the-Middle attack. What information can the attacker see?

  3. A switch's CAM table shows: Port 3: AA:BB:CC:DD:EE:FF → 192.168.1.1 and Port 7: AA:BB:CC:DD:EE:FF → 192.168.1.1 — same MAC on two ports. What does this indicate?
📌 Conclusion

ARP's lack of authentication is a foundational flaw in LAN security. ARP spoofing enables the most powerful network attacks: MitM, session hijacking, and credential theft. Dynamic ARP Inspection and 802.1X are essential enterprise controls.

🔀 Routing, Switching & Devices
Topic 45

Routing & Switching

🔀

Switching vs Routing

🔀 Switching (Layer 2)

  • Forwards frames based on MAC address
  • Operates within a single network (LAN/VLAN)
  • Uses MAC address table (CAM table)
  • Very fast — hardware-based forwarding (ASICs)
  • Creates collision domains per port (vs hub)
  • VLANs divide a switch into multiple logical networks

🌐 Routing (Layer 3)

  • Forwards packets based on IP address
  • Operates between different networks
  • Uses routing table (longest prefix match)
  • Slower than switching (but fast on modern hardware)
  • Connects LAN to WAN / internet
  • NAT, ACLs, QoS applied at router

Routing Table & Path Selection

ROUTING TABLE
# Router routing table (Linux: ip route show):
Destination      Gateway        Interface  Metric  Protocol
0.0.0.0/0        203.0.113.1    eth0       100     static   ← Default route (internet)
10.0.0.0/8       192.168.1.254  eth1       20      OSPF     ← Internal network
192.168.1.0/24   0.0.0.0        eth1       0       direct   ← Directly connected
10.1.0.0/24      10.0.0.2       eth1       30      OSPF     ← Remote subnet via OSPF

# Routing decision: longest prefix match wins
Packet to 10.1.5.100:
  10.0.0.0/8    matches (8 bits)
  10.1.0.0/24   matches (24 bits) ← MORE SPECIFIC = WINS
  → Send via 10.0.0.2

# Routing protocols:
Static   → Manually configured. Simple, secure. Doesn't adapt.
RIP      → Distance vector, max 15 hops. Legacy.
OSPF     → Link-state. Used inside companies (IGP). Fast convergence.
BGP      → Path vector. THE protocol of the internet. ISP interconnection.

# BGP Hijacking:
Attacker announces more-specific routes for victim's IP space
Other routers prefer the more-specific route → traffic diverted
2010: Pakistan Telecom hijacked YouTube globally via BGP

VLANs — Virtual LANs

Definition
A VLAN (Virtual LAN) logically segments a physical switch into multiple isolated broadcast domains. Devices in VLAN 10 cannot communicate with VLAN 20 without passing through a router (inter-VLAN routing). This enables network segmentation without physical separation.
🔐
VLANs for Security Segmentation: Finance VLAN cannot reach IoT VLAN. Guest WiFi VLAN cannot reach corporate servers. Each VLAN passes through a firewall for inter-VLAN traffic inspection. Standard enterprise security architecture.

📌 Conclusion

Switches and routers are the backbone of every network — understanding their operation is essential for designing secure topologies, placing firewalls effectively, and detecting routing anomalies that may indicate an attack in progress.

Topic 47

NAT (Network Address Translation)

🔄

Definition

Definition
NAT (Network Address Translation) allows multiple devices with private IP addresses to share a single public IP address when accessing the internet. The router maintains a translation table mapping (private IP:port) ↔ (public IP:port). Without NAT, we would have run out of IPv4 addresses far sooner.

How NAT Works

NAT TRANSLATION TABLE
# Home network with NAT:
Internal devices (private IPs):    Public IP: 203.0.113.5
  192.168.1.10 (Alice's PC)
  192.168.1.11 (Bob's Phone)
  192.168.1.12 (Smart TV)

# NAT Translation Table (router maintains this):
Inside Local      Inside Global       Outside Global
192.168.1.10:4321 → 203.0.113.5:8001 → google.com:443
192.168.1.11:5678 → 203.0.113.5:8002 → youtube.com:443
192.168.1.12:9012 → 203.0.113.5:8003 → netflix.com:443

# When google.com replies to 203.0.113.5:8001:
Router looks up table → forwards to 192.168.1.10:4321

# NAT Types:
Static NAT   → 1:1 mapping (specific internal IP always maps to specific public IP)
             → Used for servers in DMZ (always reachable at same public IP)
Dynamic NAT  → Pool of public IPs, assigned as needed
PAT (Port Address Translation) → Many:1 (most common — home routers)
             → Also called NAT Overload or NAPT

# Security implications of NAT:
+ Hides internal topology from internet (obscurity)
+ Blocks unsolicited inbound connections (implicit firewall)
- Not a firewall replacement — outbound connections can carry malware
- NAT traversal techniques (STUN/TURN) bypass NAT for P2P
- IPv6 makes NAT unnecessary — security must come from firewall

Port Forwarding

⚠️
Port Forwarding: Configures NAT router to forward all traffic on a public port to a specific internal device. E.g., public:3389 → 192.168.1.50:3389 exposes RDP to the internet. This is a high-risk configuration — attackers scan the entire internet for port 3389 (Shodan shows millions of exposed RDPs). Only forward necessary ports, use VPN instead.
🔬 Exercise
  1. How many public IP addresses does a typical home network need? Without NAT, how many would it need?
  2. Why should you use a VPN rather than port-forwarding RDP for remote access to your home computer?
📌 Conclusion

NAT extended IPv4's lifetime by decades and provides some security through obscurity. But it is NOT a firewall. Always pair NAT with a proper firewall, minimise port forwarding, and avoid exposing services directly to the internet — use VPN for remote access instead.

📦 Packet Flow & Troubleshooting
Topic 49

Packet Flow Process

📦

Complete Packet Journey

When you send an email from your laptop to a colleague's server, here's what happens at every level:

PACKET FLOW EXAMPLE: Sending email (SMTP)
# You click SEND in Outlook. Destination: mail.corp.com (10.5.0.10)

STEP 1 – Application Layer (L7)
  Outlook creates SMTP message: EHLO, MAIL FROM, RCPT TO, DATA
  Content: email headers + body + attachments

STEP 2 – Transport Layer (L4)
  TCP adds header: SRC_PORT=49152 DST_PORT=25
  Performs 3-way handshake first
  Breaks data into segments if needed

STEP 3 – Network Layer (L3)
  IP adds header: SRC_IP=192.168.1.10 DST_IP=10.5.0.10
  Determines next hop: default gateway 192.168.1.1

STEP 4 – Data Link Layer (L2)
  ARP: "Who has 192.168.1.1?" → gets MAC AA:BB:CC:11:22:33
  Ethernet frame: SRC_MAC=your_MAC DST_MAC=AA:BB:CC:11:22:33

STEP 5 – Physical Layer (L1)
  Frame encoded as electrical signals / WiFi radio waves
  Transmitted to switch port

====== At the switch ======
SWITCH: reads frame DST_MAC → looks up CAM table → forwards to router port

====== At the router (gateway) ======
ROUTER: strips L2 frame, reads L3 IP header
Routing table lookup: 10.5.0.10 → route via WAN interface
NAT: replaces 192.168.1.10 with public IP 203.0.113.5
New frame: adds new L2 header with next-hop MAC
Transmitted to ISP

====== Multiple hops across internet (BGP routing) ======
Each router: decapsulate L2, read L3, look up route, re-encapsulate L2

====== At destination mail server ======
Physical → Data Link → Network → Transport → Application
SMTP server: receives email, stores in mailbox

Security Analysis at Each Hop

HopWhat's VisibleSecurity ControlAttack Possible
Your LAN switchAll Layer 2 framesPort security, DAI, 802.1XARP spoof, MAC flood
Your router/firewallIP headers, ports, connection stateFirewall rules, NAT, IPSIP spoofing, port scanning
ISP networkAll traffic (if unencrypted)ISP filters, BCP38Traffic interception, BGP hijack
Destination firewallIP, ports, application dataPerimeter firewall, WAFPort-based bypass via 443
Destination serverFull application payloadApp security, TLS terminationXSS, SQLi, brute force

📌 Conclusion

Understanding packet flow transforms abstract theory into concrete security thinking. Every "hop" in the journey is a potential interception point. Encryption (TLS) protects the payload; IP headers and DNS queries may still leak metadata to network observers.

Attack Scenario: Phishing → Lateral Movement

ATTACK & DETECTION WALKTHROUGH
Day 1: Alice (VLAN 10, 10.1.0.50) receives phishing email
  → Opens malicious attachment
  → Malware establishes C2 connection to evil.com:443

NGFW detects: 10.1.0.50 connecting to evil.com (threat intel feed)
→ SIEM ALERT: Suspicious outbound connection from corporate VLAN
→ NGFW blocks C2 connection

Malware tries DNS tunnelling to bypass:
  Queries: aGVsbG8.evil.com, d29ybGQ.evil.com (base64 encoded)
→ DNS monitoring detects: unusually long subdomains from 10.1.0.50
→ DNS blocked on Alice's device, SIEM alert

Attacker pivots: tries ARP spoof to reach Finance VLAN 20
  Sends gratuitous ARP from 10.1.0.50
→ DAI (Dynamic ARP Inspection) blocks it — IP not in DHCP snooping table

Attacker tries to scan Finance VLAN 20 directly
  VLAN 10 to VLAN 20 requires passing internal firewall
→ Internal firewall: no rule allows workstation → Finance VLAN = BLOCKED
→ SIEM ALERT: ACL deny on internal firewall from 10.1.0.50

SOC investigates Alice's machine → isolates → remediation
Attacker never reached Finance data.

Lesson: Defence in depth + segmentation contained the breach.

📌 Conclusion

Real-world network security is about making each attack layer harder than the last. Segmentation, monitoring, and least-privilege access rules together mean that even a successful phishing attack — which is nearly impossible to prevent entirely — becomes a contained, detectable, and reversible incident rather than a catastrophic breach.

Quick Reference Glossary (30 Key Terms)

LANLocal network, one building. Fast, private. ARP spoof risk.
WANWide network, global. The internet is the biggest WAN. BGP hijack risk.
Star TopologyAll devices to central switch. Most common. Switch = single point of failure.
Mesh TopologyEvery node connected to every other. Maximum redundancy. Used in WAN/military.
OSI Model7-layer reference model. Each layer = protocol + attack type. Know all 7.
TCP/IP Model4-layer practical model. Application, Transport, Internet, Network Access.
TCPReliable, ordered, connection-oriented. SYN/SYN-ACK/ACK handshake. Ports 0–65535.
UDPFast, unreliable, connectionless. Used for DNS, VoIP, gaming. DoS amplification.
IPv432-bit address. 4.3B addresses (exhausted). Private: 10.x, 172.16.x, 192.168.x
IPv6128-bit address. 340 undecillion. Format: 2001:db8::1. SLAAC privacy risk.
SubnettingDivide network into segments. /24 = 254 hosts. Isolates breaches.
CIDR/prefix notation for subnet mask. /24 = 255.255.255.0. Usable = 2^(32-n)-2
DNSTranslates domain → IP. Port 53 UDP. Attack: cache poisoning, tunnelling.
DHCPAuto-assigns IP config. DORA process. Port 67/68. Rogue server = MitM.
ARPMaps IP → MAC on LAN. No authentication. Spoofing = MitM. Fix: DAI.
MAC Address48-bit hardware ID. Layer 2. Can be spoofed. First 3 bytes = manufacturer (OUI).
SwitchLayer 2 device. Uses CAM table (MAC→port). VLAN segmentation. MAC flood risk.
RouterLayer 3. Routes by IP. Routing table. Connects LANs + internet. BGP at ISP scale.
VLANVirtual LAN on one switch. Segment traffic. Isolate Finance, IoT, Guest. VLAN hop attack.
NATMany private IPs → one public IP. PAT most common. Hides topology. Not a firewall.
FirewallFilters traffic by rules. Stateful tracks connections. NGFW = Layer 7 inspection.
IDS/IPSIDS=detect only. IPS=detect+block. Signature vs anomaly. Snort, Suricata.
DMZSemi-trusted zone between internet and internal. Houses public-facing servers.
DNSSECSigns DNS records. Prevents cache poisoning. Use DoH for query privacy.
WPA3Latest WiFi security. SAE replaces PSK. Resists offline dictionary attacks.
Port16-bit endpoint. 0-1023 well-known. 80=HTTP, 443=HTTPS, 22=SSH, 3389=RDP.
BGPInternet routing protocol. Hijacking = redirect traffic to attacker. Pakistan Telecom 2010.
802.1XPort authentication before network access. Prevents rogue devices. Requires RADIUS.
SIEMCorrelates security logs from all devices. Detects attack patterns. Splunk, Sentinel.
HoneypotFake decoy system. Any connection = attacker alert. Cowrie, Canarytokens.

Key Formulas & Relationships

UNIT 3 FORMULAS
# Subnetting:
Usable Hosts     = 2^(32 - prefix) - 2
Number of Subnets= 2^(borrowed bits)
Subnet Mask      = 32 consecutive 1s then 0s (e.g. /24 = 255.255.255.0)

# OSI Layer → PDU → Device:
L7-L5 Application → Data     → Proxy, Load Balancer, Firewall (L7)
L4    Transport   → Segment  → Firewall (stateful)
L3    Network     → Packet   → Router, L3 Switch
L2    Data Link   → Frame    → Switch, Bridge, AP
L1    Physical    → Bit      → Hub, Cable, NIC, Repeater

# Defence in Depth layers:
CDN → Perimeter FW → DMZ → Internal FW → VLAN → NAC → SIEM → EDR

# TCP/IP encapsulation:
App Data → [TCP hdr | Data] → [IP hdr | TCP | Data] → [ETH hdr | IP | TCP | Data | ETH tail]

# Attack layer mapping:
Physical tap        → L1
ARP spoof/MAC flood → L2
IP spoof, BGP hijack→ L3
SYN flood, port scan→ L4
Session hijack      → L5
SSL strip           → L6
XSS, SQLi, phishing → L7

📌 Unit 3 Final Takeaway

Networking is the skeleton of every digital system — and every bone can be broken. From physical layer tapping to BGP hijacking, attackers operate at every level of the OSI model. A great network security architect doesn't just know the protocols; they understand how each protocol can be abused and designs networks that detect, contain, and recover from breaches. The goal isn't to prevent all attacks — it's to ensure no single attack becomes a catastrophe.
